# Инструкция по публикации поста

## 1. Создай файл статьи

Создай новый `.md` файл в папке `src/content/posts/`:

```
src/content/posts/название-поста.md
```

**Правила именования файла:**
- Только строчные буквы латиницей
- Слова через дефис: `my-docker-guide.md`
- Без пробелов и спецсимволов
- Имя файла становится URL: `/posts/my-docker-guide`

---

## 2. Заполни frontmatter

Каждая статья начинается с блока метаданных между `---`:

```markdown
---
title: "Заголовок статьи"
date: 2025-03-15
excerpt: "Краткое описание для превью и SEO. 1-2 предложения."
tags: ["docker", "linux", "ci-cd"]
category: "devops"
readTime: 8
featured: false
draft: false
---
```

### Поля frontmatter

| Поле       | Обязательное | Описание |
|------------|:---:|---|
| `title`    | ✅ | Заголовок статьи |
| `date`     | ✅ | Дата публикации в формате `YYYY-MM-DD` |
| `excerpt`  | ✅ | Краткое описание (используется в превью и SEO) |
| `tags`     | ✅ | Массив тегов: `["docker", "linux"]` |
| `readTime` | ✅ | Время чтения в минутах (целое число) |
| `category` | ❌ | Категория статьи (slug, например `ci-cd`) |
| `featured` | ❌ | `true` — показывать первой на главной (по умолчанию `false`) |
| `draft`    | ❌ | `true` — скрыть статью из публикации (по умолчанию `false`) |
| `ogImage`  | ❌ | URL изображения для Open Graph (превью в соцсетях) |

---

## 3. Напиши контент

После frontmatter пиши обычный Markdown:

```markdown
---
title: "Пример статьи"
date: 2025-03-15
excerpt: "Описание"
tags: ["linux"]
readTime: 5
---

## Введение

Обычный текст. **Жирный**, *курсив*, `инлайн-код`.

## Блоки кода

Блоки кода с подсветкой синтаксиса:

```bash
docker compose up -d
```

```yaml
services:
  app:
    image: nginx:alpine
    ports:
      - "80:80"
```

## Списки

- Пункт 1
- Пункт 2
  - Вложенный пункт

1. Шаг первый
2. Шаг второй

## Ссылки и изображения

[Текст ссылки](https://example.com)

![Alt текст](/images/screenshot.png)
```

---

## 4. Черновик vs публикация

Чтобы работать над статьёй, не публикуя её:

```markdown
---
draft: true   ← статья не появится на сайте
---
```

Когда статья готова — смени на `draft: false` или удали поле.

---

## 5. Проверь локально

Запусти dev-сервер и проверь статью в браузере:

```bash
npm run dev
```

Открой `http://localhost:4321` и найди свою статью.

---

## 6. Опубликуй

Добавь файл в git и запушь в основную ветку:

```bash
git add src/content/posts/название-поста.md
git commit -m "post: добавлена статья про название-поста"
git push origin main
```

Cloudflare Pages автоматически пересоберёт и задеплоит сайт.
Обычно деплой занимает **1-2 минуты**.

## Частые вопросы

**Как добавить новую категорию?**
Просто укажи любое значение в поле `category`. Страница категории создастся автоматически.

**Как добавить новый тег?**
Добавь строку в массив `tags`. Страница тега создастся автоматически.

**Статья не появляется на сайте?**
Проверь: `draft: false` (или поле отсутствует) и дата не в будущем.

**Как посчитать readTime?**
Примерно: количество слов / 200. Для технических статей с кодом добавь 2-3 минуты.
